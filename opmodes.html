OpModes

There are various types of OpModes in FTC Coding. However, for the beginner part of this, we will only be focusing on two types of OpModes: OpMode and LinearOpMode

OpMode

The base OpMode is the most simple of the OpModes, containing two methods init() and loop(). For example, if you had a TeleOp OpMode file called Manual labeled "manual" in the driver station, it would look like this: 

            
@TeleOp(name = "manual")
public class Manual extends OpMode {
    @Override
    public void init() {
        //Code to be run during initialize here
    }
    
    @Override
    public void loop() {
        //Code to be run every loop here
    }
}

LinearOpMode

LinearOpMode is a another basic OpMode. However, this consists of just one method, runOpMode(), rather than two methods, init() and loop(). An example of this as a TeleOp file called Manual labeled "manual" in the driver station would look like this: 

                
@TeleOp(name = "manual")
public class Manual extends LinearOpMode {
    @Override
    public void runOpMode() {
        //Enter initialize code here

        waitForStart(); //Doesn't move past this line until start button is pressed
        while (opModeIsActive()) {
            //This code loops while the op mode is active,
            //which is until the stop button is pressed
        }
    }
}
